## CONNECTION DETAILS
- **Host**: spark-thrift (Docker container service name)
- **Port**: 10000 (Spark Thrift Server)
- **Catalog**: my_catalog
- **Schema**: analytics
- **Method**: Thrift Protocol
- **DBT Adapter**: dbt-spark (version 1.5.2)

## DIRECTORY STRUCTURE
```
.
├── .dbt/
│   └── profiles.yml                 # DBT connection configuration
├── dbt/
│   ├── Dockerfile                   # DBT container image
│   ├── dbt_project.yml              # DBT project configuration
│   ├── models/
│   │   └── example_model.sql        # Sample model to test connection
│   └── seeds/                       (optional)
│   └── tests/                       (optional)
├── docker-compose.yml               # Docker Compose configuration
└── code.txt                         # This file

## QUICK START

1. Start all services:
   docker-compose up -d

2. Access dbt container:
   docker-compose exec dbt bash

3. Test connection:
   dbt debug

4. Run dbt models:
   dbt run

5. View dbt documentation:
   dbt docs generate
   dbt docs serve

## DBT PROFILES CONFIGURATION
Location: .dbt/profiles.yml

```yaml
analytics_profile:
  outputs:
    spark_thrift:
      type: spark
      method: thrift
      schema: analytics
      catalog: my_catalog
      host: spark-thrift
      port: 10000
      user: root
      threads: 4
      timeout_seconds: 300
      retries: 3
  target: spark_thrift
```

spark-thrift:
    image: apache/spark:3.4.1
    container_name: spark-thrift
    user: root
    depends_on:
      - spark-master
      - minio
    ports:
      - "10000:10000"
    environment:
      SPARK_HOME: /opt/spark
    # We use a shell command to download JARs first, then start the server
    entrypoint: /bin/bash
    command: >
      -c "
      echo 'Downloading Iceberg and AWS dependencies...' &&
      curl -s -L -o /opt/spark/jars/iceberg-spark-runtime-3.4_2.12-1.4.3.jar https://repo1.maven.org/maven2/org/apache/iceberg/iceberg-spark-runtime-3.4_2.12/1.4.3/iceberg-spark-runtime-3.4_2.12-1.4.3.jar &&
      curl -s -L -o /opt/spark/jars/hadoop-aws-3.3.4.jar https://repo1.maven.org/maven2/org/apache/hadoop/hadoop-aws/3.3.4/hadoop-aws-3.3.4.jar &&
      curl -s -L -o /opt/spark/jars/aws-java-sdk-bundle-1.12.262.jar https://repo1.maven.org/maven2/com/amazonaws/aws-java-sdk-bundle/1.12.262/aws-java-sdk-bundle-1.12.262.jar &&
      echo 'Dependencies downloaded. Starting Thrift Server...' &&
      /opt/spark/bin/spark-submit \
      --class org.apache.spark.sql.hive.thriftserver.HiveThriftServer2 \
      --master spark://spark-master:7077 \
      --conf spark.sql.extensions=org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions \
      --conf spark.sql.catalog.my_catalog=org.apache.iceberg.spark.SparkCatalog \
      --conf spark.sql.catalog.my_catalog.type=hadoop \
      --conf spark.sql.catalog.my_catalog.warehouse=s3a://dmart/ \
      --conf spark.hadoop.fs.s3a.endpoint=http://minio:9000 \
      --conf spark.hadoop.fs.s3a.access.key=admin \
      --conf spark.hadoop.fs.s3a.secret.key=password123 \
      --conf spark.hadoop.fs.s3a.path.style.access=true \
      --conf spark.hadoop.fs.s3a.impl=org.apache.hadoop.fs.s3a.S3AFileSystem \
      --conf spark.hadoop.fs.s3a.connection.ssl.enabled=false \
      --conf spark.hadoop.hive.server2.thrift.bind.host=0.0.0.0 \
      --conf spark.hadoop.hive.server2.thrift.port=10000
      "
